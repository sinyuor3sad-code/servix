# اتفاقية مستوى الخدمة (SLA) — SERVIX

**الإصدار:** 1.0 | **تاريخ السريان:** أبريل 2026

---

## 1. التزامات الأداء

| المقياس | الالتزام | طريقة القياس |
|---------|----------|-------------|
| **Uptime** | 99.9% شهرياً (≤ 43.8 دقيقة توقف/شهر) | Uptime Kuma + Grafana |
| **API Response Time (p95)** | < 500ms | Prometheus http_request_duration |
| **API Response Time (p99)** | < 1000ms | Prometheus http_request_duration |
| **Error Rate** | < 0.1% | Prometheus error rate counter |
| **Recovery Point Objective (RPO)** | < 1 ساعة | PostgreSQL streaming replication |
| **Recovery Time Objective (RTO)** | < 30 دقيقة | Automated failover script |

## 2. تعويضات التوقف

| Uptime الشهري | رصيد الخدمة |
|--------------|-------------|
| 99.0% — 99.9% | 10% من الاشتراك الشهري |
| 95.0% — 99.0% | 25% من الاشتراك الشهري |
| < 95.0% | 50% من الاشتراك الشهري |

> رصيد الخدمة يُطبق تلقائياً على فاتورة الشهر التالي

## 3. الاستثناءات

لا تُحسب الحالات التالية ضمن التوقف:
- **الصيانة المُجدولة** (إشعار 48 ساعة مسبق، الجمعة 2-4 صباحاً)
- **أحداث القوة القاهرة** (كوارث طبيعية، حرب)
- **توقف خدمات الطرف الثالث** (ZATCA API، Moyasar، CloudFlare)
- **هجمات DDoS** تتجاوز قدرة البنية التحتية
- **تعديلات طلبها العميل** تسببت في المشكلة

## 4. أولويات الدعم

| المستوى | التعريف | وقت الاستجابة | وقت الحل |
|---------|---------|-------------|----------|
| P1 — حرج | النظام بالكامل معطل | 15 دقيقة | 4 ساعات |
| P2 — عالي | خدمة رئيسية معطلة | 30 دقيقة | 8 ساعات |
| P3 — متوسط | تدهور أداء / خلل وظيفي | 2 ساعة | 24 ساعة |
| P4 — منخفض | طلب تحسين / سؤال | يوم عمل | 5 أيام عمل |

## 5. التحقق والشفافية

### المراقبة المستمرة
- **Uptime Kuma**: فحص كل 60 ثانية لكل الخدمات
- **Grafana Dashboards**: 6+ لوحات عامة
- **Prometheus Alerts**: تنبيهات فورية عند تجاوز العتبات
- **SLO Dashboard**: error budget tracking مستمر

### التقارير
- **أسبوعياً**: تقرير أداء تلقائي (cron)
- **شهرياً**: تقرير SLA مع uptime + latency + incidents
- **ربع سنوياً**: اختبار DR + تقرير النتائج

## 6. التواصل أثناء الحوادث

1. **تنبيه فوري** عبر SMS/WhatsApp للمستأجرين المتأثرين
2. **صفحة حالة** تُحدث كل 15 دقيقة أثناء الحادث
3. **Post-mortem** خلال 48 ساعة من الحل
4. **تقرير جذري** يُشارك مع المستأجرين المتأثرين
