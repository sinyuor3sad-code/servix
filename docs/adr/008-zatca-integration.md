# ADR-008: ZATCA Phase 2 Integration

## التاريخ
2026-03-25

## الحالة
مقبول

## السياق
يجب الامتثال لمتطلبات هيئة الزكاة والضريبة والجمارك (ZATCA) للفوترة الإلكترونية Phase 2 (التكامل والربط).

## القرار
**تكامل مباشر** مع ZATCA API باستخدام Node.js crypto للتوقيع و UBL 2.1 XML builder مخصص.

## البدائل المدروسة

| البديل | المميزات | العيوب |
|--------|----------|--------|
| مكتبة ZATCA جاهزة (fatoorajs) | سرعة التنفيذ | تبعية خارجية، قد تتأخر في التحديث |
| **تكامل مباشر** ✅ | تحكم كامل، بدون تبعيات | جهد تطوير أكبر |
| وسيط خارجي (Cleartax) | سهولة الاستخدام | تكلفة شهرية، vendor lock-in |

## التفاصيل التقنية
- CSR generation: ECDSA secp256k1
- XML: UBL 2.1 مع كل الحقول المطلوبة
- التوقيع: XML-DSIG (SHA-256 + ECDSA)
- QR Code: TLV encoding (8 tags)
- تخزين المفاتيح: AES-256-GCM encryption

## النتائج
### إيجابية
- لا تبعيات خارجية — تحكم كامل في pipeline الفوترة
- سهولة تحديث عند تغيير متطلبات ZATCA
- اختبارات وحدة شاملة للتوقيع والـ XML

### سلبية
- جهد تطوير أولي أكبر (~2 أسابيع)
- يحتاج متابعة تحديثات ZATCA API يدوياً
